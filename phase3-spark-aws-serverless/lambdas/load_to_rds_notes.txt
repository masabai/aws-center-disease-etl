Done. Key differences from the Redshift version:

psycopg2 replaces the Redshift Data API — no polling loop needed, calls are synchronous

Credentials pulled from Secrets Manager (rds/cdc_postgres) — store host, port, dbname, username, password there

Parquet files read from S3 via pyarrow → pandas → executemany bulk insert

topic_id changed from TIMESTAMP to VARCHAR(50) — Redshift had that as a type mismatch anyway

Two things to set up on AWS before running:

Secrets Manager secret named rds/cdc_postgres with your RDS connection details

Lambda layer (or Docker image) with psycopg2-binary and pyarrow — both needed at runtime
